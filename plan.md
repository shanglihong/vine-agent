# Vine-Agent 迭代路线图与 To-Do 清单

本文档记录了 Vine-Agent 项目后续的核心功能迭代规划与实现清单。

## 📌 核心 To-Do 列表与优先级

根据系统的整体设计原则与演进规划，后续工作划分为四个级别（P0~P3）：

---

### 🔴 P0: 核心框架与基础管理（最高优先级，立即实施）

#### 1. Project（项目）上层包装
*   **目标**：在会话层之上引入项目（Project）维度，实现一个项目管理多个会话的逻辑。
*   **具体任务**：
    - [ ] **数据库变更**：在 SQLite 数据库中新建 `projects` 表（包含 `id`, `name`, `description`, `created_at`, `updated_at`, `metadata` 字段），并在 `sessions` 表中新增 `project_id` 字段建立外键关联。
    - [ ] **领域实体定义**：新建 `domain/project` 目录，并在该包下定义 `Project` 实体与 `ProjectRepository` 仓储接口，详情参考设计模式规范：[Memory 模块设计文档](doc/memory.md)。
    - [ ] **应用层编排**：在 `app/` 目录下新增 Project 管理服务，支持 Project 的增删改查及关联 Session。

#### 2. 对话生成主动中断与恢复
*   **目标**：支持在流式生成中，用户能够随时手动取消（Cancel/Abort）当前生成，并能够从截断位置继续（Continue）生成后续文本。
*   **具体任务**：
    - [x] **主动取消**：提供 Abort 接口，接收客户端取消请求，并通过 cancel context 信号中止大模型的 Stream 生成。同时将已生成的部分文本包装后写入 [session.go](domain/memory/session/session.go) 消息流中。
    - [x] **断点继续**：在 API 接口中提供 `Continue` 生成接口，加载最后一个截断的 Assistant 消息，自动补齐剩余内容。

#### 3. Session 管理强化
*   **目标**：完善 Session 的深度管理。
*   **具体任务**：
    - [ ] **级联彻底删除**：修改 [sqlite/session.go](infra/persistence/sqlite/session.go)，在删除 Session 时，物理清理 `messages` 数据，并同步执行 `messages_fts` 检索虚拟表的级联删除，防止产生数据垃圾。
    - [x] **标题自动生成**：引入后台异步任务，在会话进行到前两轮后，调用大语言模型（如 DeepSeek）从前两轮对话中提炼精炼的主题，调用 `session.go` 的 API 自动更新 Session Title。

#### 4. 个人偏好集成
*   **具体任务**：
    - [ ] **上下文优化**：仅使用对话进行提取，忽略工具调用，思考等非对话内容
    - [x] **样式改良**：使用和网络搜索结果展示一样的卡片
    - [x] **多session聚合记忆演进**

---

### 🟡 P1: 核心能力扩展（高优先级，中期计划）

#### 1. 常用工具拓展与动态 Registry 机制
*   **目标**：让大模型具备在限定范围内读取本地文件与进行网页检索的能力。
*   **具体任务**：
    - [x] **BE FE 删除mock工具**
    - [ ] **本地文件工具**：在 [infra/tools](infra/tools)（参考 [doc/tool.md](doc/tool.md)）中实现 `ReadFilesTool`, `WriteFileTool`, `ListDirTool`。
    - [x] **Web 检索工具**：实现网页爬取与信息搜集工具，用于获取外部最新资讯。
    - [ ] **动态 Registry 注册表**：在 `domain/tool` 引入 `Registry` 服务，支持按照 Project 级别的元配置动态注入和启用所需的工具集合。

#### 2. 事件驱动解耦异步任务
*   **目标**：提高主对话流接口响应速度，优化体验。
*   **具体任务**：
    - [ ] **Pub/Sub 改造**：结合 [doc/event.md](doc/event.md) 设计，在用户发送消息、流式生成结束等关键节点发布事件（如 `EventMessageCreated`），通过后台异步订阅者处理“用户画像演进”、“Session标题生成”等高耗时业务逻辑。

#### 3. 智能上下文窗口管理 (Context Slider)
*   **目标**：保障长对话的稳定性，防止大模型 Context Window 溢出。
*   **具体任务**：
    - [ ] **滑动窗口摘要**：当 Session 的消息累计 Token 接近大模型窗口限制时，自动触发后台对老旧消息段进行 Summary 压缩，将老历史提炼为系统提示词（System Prompt）输入，保留最近 N 轮的完整对话文本。

---

### 🔵 P2: 安全管控与多平台接入（中优先级，进阶计划）

#### 1. 工具安全沙箱与路径白名单限制 (Tool Sandboxing)
*   **目标**：防止大模型通过工具非法越权操作宿主机系统。
*   **具体任务**：
    - [ ] **白名单验证**：对于文件读写工具，在 `Execute` 调用前进行强校验，只允许操作当前 Project 对应的本地工作目录，拒绝任何包含 `../` 或系统敏感路径的操作。

#### 2. 长期画像冲突消解与遗忘机制
*   **目标**：精炼用户偏好特征，避免长期累积导致画像冗余或矛盾。
*   **具体任务**：
    - [ ] **遗忘与冲突解决**：在 [doc/memory.md](doc/memory.md) 描述的画像演化中，重构 `Extractor` 算法，在合并新偏好/事实时自动清洗并覆盖掉与当前陈述相反的历史旧记录。

#### 3. 多平台 IM 接入适配器
*   **目标**：提供更加多样的唤醒与使用渠道。
*   **具体任务**：
    - [ ] **IM 适配**：在基础设施层提供 Telegram Bot / Discord / Slack 接入，支持将异步任务结果（如 P0/P1 的后台任务）主动推送给用户。

---

### 🟢 P3: 高级进化与多智能体（低优先级，长期愿景）

#### 1. 技能库自主沉淀 (Self-Improving)
*   **目标**：借鉴 Nous Research 的 Hermes Agent 自主进化机制。
*   **具体任务**：
    - [ ] **技能提炼**：当 Agent 成功解决一项复杂代码或文档任务后，能够提取其调用工具的链条及具体解决代码，保存为独立的自定义 Skill，在后续相同场景下直接复用。

#### 2. 物理隔离的代码执行沙箱 (Docker/WASM)
*   **目标**：实现安全的动态代码运行。
*   **具体任务**：
    - [ ] **沙箱环境**：引入轻量级 Docker 或 WASM 隔离环境。大模型生成的测试脚本、物理命令必须在沙箱中运行并捕获 `stdout/stderr` 返回，以提供完全安全的运行期验证（如一键运行 `go test` 并修复编译错误）。

#### 3. 混合语义检索 (Vector RAG)
*   **目标**：克服传统关键词匹配（FTS5）的局限，提供语义检索。
*   **具体任务**：
    - [ ] **嵌入式向量数据库**：引入嵌入式向量存储（如 `SQLite-vec`），将知识库文档和历史对话消息进行向量表示，提供“向量语义+关键词”的双重混合召回（Hybrid Search）。

#### 4. 人机双向互动调试 (Interactive HITL)
*   **目标**：在遇到阻塞性问题时能够实时协同人类排查。
*   **具体任务**：
    - [ ] **协同调试流**：在 Agent 执行编译或自动化测试失败时，支持挂起流程，整理报错信息并提供若干条修复选项（或提供交互终端）推送给前端，等待用户确认修复后再继续任务。
